Introducción a Unix/Bash: Guía práctica con ejercicios

Autor/a

Roberto Álvarez

1 Introducción a Unix/Bash

1.1 Cómo usar este material

  • Espacio de trabajo seguro: Crea y trabaja siempre dentro de un directorio de práctica dedicado.
  • Ejecución consciente: Copia/pega comandos con atención. Lee antes de ejecutar.
  • Metodología activa: En cada ejercicio, escribe el comando y una breve explicación de lo que esperas ver (predicción → ejecución → verificación).
  • Autocompletado: Usa la tecla TAB para completar rutas y nombres de archivos.
  • Ayuda: Usa man comando para consultar el manual (presiona q para salir).

1.2 Objetivos de aprendizaje

Al finalizar esta guía, podrás:

  • Navegar eficientemente por el sistema de archivos (pwd, ls, cd).
  • Gestionar archivos y directorios (mkdir, cp, mv, rm).
  • Inspeccionar y explorar archivos de texto (cat, less, head, tail, wc).
  • Buscar patrones con grep y expresiones regulares básicas.
  • Encadenar comandos mediante tuberías (|) y redirecciones (>, >>).
  • Utilizar comodines (wildcards) de forma efectiva.
  • Comprimir y descomprimir archivos y directorios (tar, gzip).
  • Realizar transformaciones básicas de texto con sed y awk.

1.3 Convenciones tipográficas

  • $ indica un comando que se debe escribir en la terminal (no se incluye el prompt real).
  • archivo.txt, directorio/ son ejemplos; reemplázalos por tus rutas reales.
  • Los bloques de código muestran comandos y su salida esperada.
  • TAB representa la tecla de autocompletado.
Seguridad básica (¡Muy importante!)
  • Cuidado con rm -r: Evita usarlo hasta entender perfectamente qué elimina.
  • Verifica antes de borrar: Usa pwd para confirmar tu ubicación y ls para listar el contenido.
  • Espacio de práctica: Realiza los ejercicios de eliminación solo en tu directorio de práctica.
  • Copia de seguridad: Considera hacer copias de archivos importantes antes de modificarlos o eliminarlos.

3 Gestión de archivos y directorios

3.1 Comandos de gestión

Comando Descripción Ejemplo
touch Crea un archivo vacío o actualiza su marca de tiempo. $ touch archivo.txt
cp Copia archivos o directorios. $ cp origen destino
mv Mueve o renombra archivos/directorios. $ mv viejo nuevo
rm Elimina archivos (¡Cuidado! No va a la papelera). $ rm archivo.txt

  • Usa -v (verbose) para que los comandos muestren qué están haciendo.

    cp -v origen destino

3.3 Búsqueda de patrones con grep

grep es una herramienta poderosa para buscar líneas que coincidan con un patrón (texto simple o expresión regular) en uno o varios archivos.

3.3.1 Opciones comunes de grep

Opción Descripción
-i Búsqueda insensible a mayúsculas/minúsculas.
-w Coincide solo con palabras completas.
-c Cuenta el número de líneas que coinciden.
-n Muestra el número de línea de cada coincidencia.
-v Invierte la búsqueda (muestra líneas que NO coinciden).
-l Solo lista los nombres de los archivos que contienen coincidencias.
-A N Muestra N líneas después de la coincidencia (After).
-B N Muestra N líneas antes de la coincidencia (Before).
-C N Muestra N líneas antes y después de la coincidencia (Context).

Crea un archivo sistemas.txt con el siguiente contenido para los ejercicios:

$ cat > sistemas.txt << 'EOF'

UNIX es poderoso.
Este texto no tiene la palabra.

El sistema operativo macOS está basado en UNIX.

3.3.4 Expresiones regulares básicas con grep

grep acepta expresiones regulares (regex) para búsquedas más complejas.

Patrón Significado Ejemplo (coincide con)
^ Inicio de línea. ^Unix (líneas que comienzan con “Unix”)
$ Fin de línea. .$ (líneas que terminan con un punto)
. Cualquier carácter (excepto salto de línea). s.ste.a (“sistema”, “sisteia”, etc.)
* Cero o más repeticiones del carácter anterior. sist*ema (“sistema”, “sisttema”, etc.)
[ ] Conjunto de caracteres. [Uu]nix (“Unix” o “unix”)
[^ ] Negación del conjunto. [^0-9] (cualquier carácter que no sea un dígito)

Ejercicio 6.6: Regex - Inicio de línea

Ejercicio 6.7: Regex - Fin de línea

Busca líneas que terminen con un punto.

Ejercicio 6.8: Regex - Dígitos

Busca líneas que contengan al menos un dígito.

grep "[0-9]" sistemas.txt

4 Redirecciones y tuberías (Pipes)

Conceptos clave

Salida estándar (stdout): Donde los comandos envían sus resultados por defecto (normalmente la pantalla).

Redirección: Capturar la salida de un comando y enviarla a un archivo.

>> redirige la salida a un archivo (sobrescribe) 
>> redirige la salida añadiendo al archivo 
| envía la salida de un comando como entrada a otro

Ejemplos

# Guardar la lista de archivos en un archivo
ls -la > listado_completo.txt
# Añadir la fecha al final de un archivo
$ date >> listado_completo.txt
# Filtrar la lista para mostrar solo archivos .txt
$ ls -la | grep ".txt"
# Contar cuántos archivos .txt hay
$ ls *.txt | wc -l

4.1 Comodines (Wildcards)

Los comodines permiten seleccionar múltiples archivos/directorios basándose en patrones en sus nombres.

Comodín Uso Ejemplo
* Coincide con cualquier número de caracteres (incluyendo ninguno). *.txt (todos los archivos .txt)
? Coincide con exactamente un carácter. documento?.pdf (documento1.pdf, documentoA.pdf)
[ ] Coincide con cualquier carácter dentro del conjunto. [aeiou]* (archivos que comienzan con vocal)
[! ] o [^ ] Coincide con cualquier carácter que no esté en el conjunto. [!0-9]* (archivos que no comienzan con dígito)

5 Compresión y descompresión

5.1 Herramientas principales

gzip / gunzip: Comprime/descomprime archivos individuales (extensión .gz). tar: Agrupa múltiples archivos/directorios en un solo archivo (tarball). Combinado con gzip (.tar.gz o .tgz) permite comprimir.

Uso común

Tarea Comando
Comprimir un archivo (manteniendo original) gzip -k archivo.txt → archivo.txt.gz
Descomprimir un archivo .gz gunzip archivo.txt.gz o gzip -d archivo.txt.gz
Crear un tarball comprimido de un directorio tar -czvf archivo.tar.gz directorio/
Listar contenido de un tarball tar -tzvf archivo.tar.gz
Extraer un tarball comprimido tar -xzvf archivo.tar.gz

6 Herramientas avanzadas de procesamiento de texto

sed (stream editor)

sed realiza transformaciones básicas en un flujo de texto (línea por línea).

Uso común Comando
Sustituir texto (reemplazar) sed 's/patrón/reemplazo/g' archivo
Eliminar líneas sed '/patrón/d' archivo
Mostrar un rango de líneas sed -n '5,10p' archivo
Insertar texto antes/después de una línea sed '/patrón/i\Texto antes' archivo (antes)<br>sed '/patrón/a\Texto después' archivo (después)

Ejemplo práctico: Cambiar todas las ocurrencias de “Windows” por “Linux” en un archivo config.txt.

sed 's/Windows/Linux/g' config.txt
Nota: Por default, sed muestra el resultado en pantalla.

Para guardar los cambios en el mismo archivo, usa la opción -i (in-place):

sed -i 's/Windows/Linux/g' config.txt

6.1 awk (Lenguaje de procesamiento de patrones)

awk es ideal para procesar datos estructurados en columnas (como CSV o TSV).

Uso común Comando
Imprimir columnas específicas awk ‘{print $1, $3}’ archivo.csv
Usar un separador diferente awk -F’,’ ‘{print $2}’ archivo.csv (separador: coma)
Filtrar líneas basadas en una condición a wk ‘$1 > 100 {print $0}’ datos.txt (si la col1 > 100)
Calcular sumas o promedios awk ‘{sum += $2} END {print sum}’ archivo.txt

Ejemplo práctico:

Dado un archivo ventas.csv con columnas Producto,Cantidad,Precio, calcular el total de ventas.

awk -F',' '{total += $2 * $3} END {print "Total: $" total}' ventas.csv

8.1 Introducción a wget y curl

En entornos Unix/Linux, wget y curl son dos herramientas fundamentales para transferir datos desde la línea de comandos. Aunque comparten funcionalidades, tienen enfoques y características diferentes:

Característica wget curl Enfoque principal Descarga recursiva de sitios web y archivos Transferencia de datos usando múltiples protocolos Protocolos HTTP, HTTPS, FTP HTTP, HTTPS, FTP, FTPS, SCP, SFTP, TFTP, LDAP, DICT, FILE, y más Recursividad Sí (descarga enlaces internos) No (por defecto) Seguimiento de redirecciones Sí (automático) Sí (con opción -L) Reanudar descargas Sí (-c) Sí (-C -) Interacción con APIs Limitada Excelente (envía datos, headers personalizados) Scripting Bueno para descargas automáticas Mejor para APIs y transferencias complejas ### wget: El descargador web

Sintaxis básica

wget [opciones] [URL]

Ejemplos comunes

  1. Descargar un archivo simple:
wget https://ejemplo.com/archivo.zip
  1. Descargar con nombre personalizado:
wget -O mi_archivo.zip https://ejemplo.com/archivo.zip
  1. Descargar en segundo plano:
wget -b https://ejemplo.com/archivo_grande.iso
  1. Reanudar una descarga interrumpida:
wget -c https://ejemplo.com/archivo_interrumpido.zip
  1. Descargar recursivamente un sitio web:
wget -r -l 2 --no-parent https://ejemplo.com/seccion/

-r: recursivo -l 2: profundidad de 2 niveles –no-parent: no subir al directorio padre

  1. Descargar todos los archivos de un tipo específico:
wget -r -A "*.pdf" https://ejemplo.com/documentos/
  1. Descargar desde una lista de URLs:
wget -i lista_urls.txt
  1. Limitar velocidad de descarga:
wget --limit-rate=100k https://ejemplo.com/archivo_grande.iso

8.1.1 curl: El cliente de transferencia de datos

Sintaxis básica

curl [opciones] [URL]

Ejemplos comunes

  1. Descargar un archivo:
curl -O https://ejemplo.com/archivo.txt
  1. Descargar con nombre personalizado:
curl -o resultado.json https://api.ejemplo.com/datos
  1. Seguir redirecciones:
curl -L https://ejemplo.com/acortado
  1. Enviar una petición GET con parámetros:
curl "https://api.ejemplo.com/buscar?q=termino&pagina=1"
  1. Enviar una petición POST con datos:
curl -X POST -d "usuario=juan&clave=secreta" https://api.ejemplo.com/login
  1. Enviar datos JSON:
curl -X POST -H "Content-Type: application/json" \
  -d '{"usuario": "juan", "email": "juan@ejemplo.com"}' \
  https://api.ejemplo.com/registro
  1. Enviar archivo como datos:
curl -X POST -F "archivo=@datos.csv" https://api.ejemplo.com/subir
  1. Usar autenticación básica:
curl -u usuario:clave https://api.ejemplo.com/protegido
  1. Enviar headers personalizados:
curl -H "Authorization: Bearer token123" \
  -H "X-API-Key: abcdef" \
  https://api.ejemplo.com/datos
  1. Guardar la respuesta HTTP completa (headers + cuerpo):
curl -i https://ejemplo.com/
  1. Solo mostrar los headers de la respuesta:
curl -I https://ejemplo.com/

## Búsqueda de patrones con `grep`

`grep` es una herramienta poderosa para buscar líneas que coincidan con
un patrón (texto simple o expresión regular) en uno o varios archivos.

### Opciones comunes de `grep`

| Opción | Descripción |
|----------------------------|--------------------------------------------|
| `-i` | Búsqueda insensible a mayúsculas/minúsculas. |
| `-w` | Coincide solo con palabras completas. |

# Compresión y descompresión

## Herramientas principales

# Herramientas avanzadas de procesamiento de texto

**`sed`** (**s**tream **ed**itor)

`sed` realiza transformaciones básicas en un flujo de texto (línea por

::: callout-tip
## Nota: Por *default*, `sed` muestra el resultado en pantalla.

(in-place):

``` bash
sed -i 's/Windows/Linux/g' config.txt
```

# Descarga de archivos desde la terminal con `wget` y `curl`

## Introducción a wget y curl

En entornos Unix/Linux, `wget` y `curl` son dos herramientas
fundamentales para transferir datos desde la línea de comandos. Aunque
comparten funcionalidades, tienen enfoques y características diferentes:

Característica `wget` `curl` Enfoque principal Descarga recursiva de
sitios web y archivos Transferencia de datos usando múltiples protocolos
Protocolos HTTP, HTTPS, FTP HTTP, HTTPS, FTP, FTPS, SCP, SFTP, TFTP,
LDAP, DICT, FILE, y más Recursividad Sí (descarga enlaces internos) No
(por defecto) Seguimiento de redirecciones Sí (automático) Sí (con
opción -L) Reanudar descargas Sí (-c) Sí (-C -) Interacción con APIs
Limitada Excelente (envía datos, headers personalizados) Scripting Bueno
para descargas automáticas Mejor para APIs y transferencias complejas
\### `wget`: El descargador web

**Sintaxis básica**

``` bash
wget [opciones] [URL]
```

Ejemplos comunes

1.  Descargar un archivo simple:

``` bash
wget https://ejemplo.com/archivo.zip
```

2.  Descargar con nombre personalizado:

``` bash
wget -O mi_archivo.zip https://ejemplo.com/archivo.zip
```

3.  Descargar en segundo plano:

``` bash
wget -b https://ejemplo.com/archivo_grande.iso
```

4.  Reanudar una descarga interrumpida:

``` bash
wget -c https://ejemplo.com/archivo_interrumpido.zip
```

5.  Descargar recursivamente un sitio web:

``` bash
wget -r -l 2 --no-parent https://ejemplo.com/seccion/
```

-r: recursivo -l 2: profundidad de 2 niveles --no-parent: no subir al
directorio padre

6.  Descargar todos los archivos de un tipo específico:

``` bash
wget -r -A "*.pdf" https://ejemplo.com/documentos/
```

7.  Descargar desde una lista de URLs:

``` bash
wget -i lista_urls.txt
```

8.  Limitar velocidad de descarga:

``` bash
wget --limit-rate=100k https://ejemplo.com/archivo_grande.iso
```

### `curl`: El cliente de transferencia de datos

**Sintaxis básica**

``` bash
curl [opciones] [URL]
```

**Ejemplos comunes**

1.  Descargar un archivo:

``` bash
curl -O https://ejemplo.com/archivo.txt
```

2.  Descargar con nombre personalizado:

``` bash
curl -o resultado.json https://api.ejemplo.com/datos
```

3.  Seguir redirecciones:

``` bash
curl -L https://ejemplo.com/acortado
```

4.  Enviar una petición GET con parámetros:

``` bash
curl "https://api.ejemplo.com/buscar?q=termino&pagina=1"
```

5.  Enviar una petición POST con datos:

``` bash
curl -X POST -d "usuario=juan&clave=secreta" https://api.ejemplo.com/login
```

6.  Enviar datos JSON:

``` bash
curl -X POST -H "Content-Type: application/json" \
  -d '{"usuario": "juan", "email": "juan@ejemplo.com"}' \
  https://api.ejemplo.com/registro
```

7.  Enviar archivo como datos:

``` bash
curl -X POST -F "archivo=@datos.csv" https://api.ejemplo.com/subir
```

8.  Usar autenticación básica:

``` bash
curl -u usuario:clave https://api.ejemplo.com/protegido
```

9.  Enviar headers personalizados:


# Proyecto final: Script de backup automatizado

Objetivo: Crear un script de Bash que realice un backup de todos los
archivos modificados en los últimos 7 días dentro de un directorio
específico.

Requisitos del script (backup_reciente.sh):

Debe recibir el directorio a respaldar como primer argumento. Crear un
archivo comprimido (tar.gz) con los archivos modificados en los últimos
7 días. El nombre del archivo de backup debe incluir la fecha (formato:
backup_YYYY-MM-DD.tar.gz). Mover el archivo de backup a un directorio
central de backups (por ejemplo, \~/backups/). Registrar la operación
(fecha, directorio respaldado, nombre del archivo) en un archivo de log
(\~/backups/backup.log). Esqueleto del script:

bash #!/bin/bash

# Verificar que se proporcionó un directorio

if \[ \$# -ne 1 \]; then echo "Uso: \$0 <directorio_a_respaldar>" exit 1
fi

DIR_ORIGEN="$1"
DIR_BACKUP="$HOME/backups" LOG="$DIR_BACKUP/backup.log"
FECHA=$(date +%Y-%m-%d)
ARCHIVO_BACKUP="$DIR_BACKUP/backup_$FECHA.tar.gz"

# Crear directorio de backups si no existe

mkdir -p "\$DIR_BACKUP"

# Crear backup (solo archivos modificados en los últimos 7 días)

find
"$DIR_ORIGEN" -type f -mtime -7 -print0 | tar -czvf "$ARCHIVO_BACKUP"
--null -T -

# Verificar éxito de la creación

if \[ \$? -eq 0 \]; then echo "Backup creado exitosamente:
$ARCHIVO_BACKUP"
    # Registrar en el log
    echo "$(date): Backup de \$DIR_ORIGEN completado -\>
$ARCHIVO_BACKUP" >> "$LOG" else echo "Error al crear el backup." \>&2
exit 1 fi Para ejecutar el script:

Guarda el código en un archivo backup_reciente.sh. Dale permisos de
ejecución: chmod +x backup_reciente.sh. Ejecútalo: ./backup_reciente.sh
/ruta/del/directorio. Conclusión
